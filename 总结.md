# Scrapy 使用水平评价

基于对整个 CnBlog 项目的深入分析，以下是当前 Scrapy 使用程度的评价。

---

## 综合评级：中高级（进阶使用者）

已经超越了入门阶段，对 Scrapy 的核心机制有较深理解，并能组合多个高级特性构建一个接近生产级的爬虫系统。

---

## 做得好的部分（亮点）

| 模块 | 评价 |
|---|---|
| **全异步架构** | 使用 `AsyncioSelectorReactor` + async 回调 + async Pipeline，这是 Scrapy 2.x 的现代用法，很多人还没切到这个模式 |
| **Pipeline 链设计** | 6 个 Pipeline 分工明确（清洗→校验→图片下载→透传→批量入库），优先级编排合理，体现了对 Item Pipeline 执行顺序的理解 |
| **批量数据库写入** | `AsyncBatchDataBasePipeline` 动态构建多行 INSERT SQL + UPSERT 去重 + 连接池 + 优雅关闭，这是工程化思维 |
| **ImagesPipeline 定制** | 重写 `file_path()`、`get_media_requests()`、`item_completed()` 三个方法，处理了 Twisted Failure，属于中高级用法 |
| **Cookie 池管理** | Redis 同步/异步双实现，SCAN 迭代、生成器模式、按账号管理，有生产环境意识 |
| **反检测登录** | Playwright 自动化登录 + webdriver 属性覆盖 + 浏览器指纹伪装 + reCAPTCHA 检测 |
| **爬虫礼仪** | AutoThrottle、随机延迟、robots.txt 遵守、域名级并发控制 |
| **自定义 Extension** | 通过信号监听实现统计汇总，理解了 Scrapy 的信号机制和扩展点 |

## 还有提升空间的部分

| 模块 | 建议 |
|---|---|
| **Item Loaders** | 目前直接 `item['field'] = value` 赋值，没有使用 `ItemLoader` + `TakeFirst`/`MapCompose`/`Join` 等处理器。对于复杂页面提取，Loader 能大幅提升代码可维护性 |
| **CrawlSpider / LinkExtractor** | 目前手动处理分页，如果爬取规则更复杂的站点，`Rule` + `LinkExtractor` 能减少大量样板代码 |
| **CSS 选择器** | 全部用 XPath，建议两者结合使用，CSS 在简单选择时更简洁 |
| **Spider 拆分** | 一个 Spider 同时处理 `www` 和 `news` 两个子站，通过 `if/else` 分支路由。建议拆成两个 Spider，各自职责更清晰 |
| **错误处理** | `parse_detail` 中有 `except Exception: pass`，会静默吞掉错误，建议至少记录日志 |
| **Cookie 刷新** | Cookie 在类定义时一次性获取，爬取过程中不会轮换或刷新。长时间爬取可能导致认证失效 |
| **代理中间件** | 已写好框架但注释掉了，生产环境通常需要代理轮换 |
| **Feed Exports** | 基本没用 Scrapy 内置的导出功能（JSON/CSV），而是自己写了 Pipeline 来实现 |

## 进阶补充：数据提取层（ItemLoader）

### 什么是 ItemLoader

ItemLoader 是 Scrapy 提供的结构化数据提取工具，它将「提取逻辑」和「数据清洗逻辑」从 Spider 中解耦出来，集中到 Item 定义或 Loader 类中。

**核心概念：**

- **Input Processor**：数据提取后立即执行，作用于每个匹配到的值（列表），结果存入 Loader 内部
- **Output Processor**：调用 `load_item()` 时执行，作用于最终字段值，结果写入 Item
- 每个字段可以独立配置不同的 Processor

### 常用内置 Processor

| Processor | 作用 | 典型场景 |
|---|---|---|
| `TakeFirst()` | 返回列表中第一个非空值 | 单值字段（标题、URL） |
| `Join(separator)` | 将列表拼接为字符串 | 多段文本合并 |
| `MapCompose(*functions)` | 链式调用多个函数，逐层处理 | 多步清洗（去空白→去标签→截断） |
| `Identity()` | 不做处理，原样返回 | 需要保留完整列表的字段 |

### 结合 CnBlog 项目的改造示例

**当前写法（Spider 中直接赋值）：**

```python
# spiders/article.py 中的现有代码
item = ArticleItem()
item['title'] = title.strip() if title else ''
item['description'] = description.strip() if description else ''
item['url'] = url
item['cover_image'] = cover_image
item['created_at'] = created_at
```

**改造后：定义 ArticleLoader**

```python
# loaders.py（新建文件）
from itemloaders import ItemLoader
from itemloaders.processors import TakeFirst, MapCompose, Join
from datetime import datetime
import pytz


def strip_text(text):
    """去除首尾空白"""
    return text.strip() if isinstance(text, str) else text


def parse_datetime(text):
    """将字符串解析为带时区的 datetime"""
    if not text:
        return None
    try:
        dt = datetime.strptime(text.strip(), '%Y-%m-%d %H:%M')
        return pytz.timezone('Asia/Shanghai').localize(dt)
    except (ValueError, AttributeError):
        return None


def normalize_image_url(url):
    """处理协议相对 URL"""
    if url and url.startswith('//'):
        return 'https:' + url
    return url


class ArticleLoader(ItemLoader):
    default_item_class = ArticleItem
    default_input_processor = MapCompose(strip_text)
    default_output_processor = TakeFirst()

    # 特殊字段单独配置
    cover_image_in = MapCompose(strip_text, normalize_image_url)

    created_at_in = MapCompose(strip_text, parse_datetime)
    created_at_out = TakeFirst()

    updated_at_in = MapCompose(strip_text, parse_datetime)
    updated_at_out = TakeFirst()

    views_in = MapCompose(lambda x: int(x) if x and x.isdigit() else 0)
    views_out = TakeFirst()
```

**Spider 中的使用方式：**

```python
# spiders/article.py 中改造 parse 方法
from cnblog.loaders import ArticleLoader

def parse(self, response):
    for article in response.xpath('//article[@class="post-item"]'):
        loader = ArticleLoader(item=ArticleItem(), selector=article)
        loader.add_xpath('title', './/a[@class="post-item-title"]/text()')
        loader.add_xpath('url', './/a[@class="post-item-title"]/@href')
        loader.add_xpath('description', './/p[@class="post-item-summary"]/text()')
        loader.add_xpath('cover_image', './/img/@data-src | .//img/@src')
        loader.add_xpath('created_at', './/span[@class="post-meta-item"]/time/text()')
        loader.add_value('updated_at', datetime.now(pytz.timezone('Asia/Shanghai')))
        yield loader.load_item()
```

### ItemLoader 的优势

1. **提取逻辑集中管理**：清洗规则定义在 Loader 中，Spider 只负责「选什么」，不负责「怎么洗」
2. **可复用**：多个 Spider 爬取同类数据时共享同一个 Loader
3. **可测试**：Processor 是纯函数，可以独立单元测试
4. **选择器绑定**：`selector=article` 让 XPath 自动基于当前节点，避免写出 `.//` 前缀的冗长路径

## 进阶补充：爬取策略层（CrawlSpider / LinkExtractor）

### 什么是 CrawlSpider

`CrawlSpider` 是 Scrapy 提供的高级 Spider 基类，通过声明式的 `Rule` + `LinkExtractor` 自动处理链接跟踪和分页，替代手动编写分页逻辑。

**核心概念：**

- **LinkExtractor**：定义「从页面中提取哪些链接」，支持正则、allow/deny 规则、restrict_css/restrict_xpath
- **Rule**：定义「提取到的链接怎么处理」——用哪个 callback 解析、是否继续跟踪
- **parse 方法被占用**：CrawlSpider 内部使用 `parse()` 来驱动 Rule 引擎，自定义逻辑必须用其他方法名

### 常用 LinkExtractor 参数

| 参数 | 作用 | 示例 |
|---|---|---|
| `allow` | 正则匹配允许的 URL | `r'/p/\d+\.html'` |
| `deny` | 正则匹配排除的 URL | `r'/tag/'` |
| `restrict_css` | CSS 选择器限定提取区域 | `'div.pager a'` |
| `restrict_xpath` | XPath 限定提取区域 | `'//div[@id="pager"]//a'` |
| `unique=True` | 去重（默认 True） | 避免重复请求同一 URL |

### 结合 CnBlog 项目的改造示例

**当前写法（手动分页）：**

```python
# spiders/article.py 中的现有分页逻辑
next_page = response.xpath(
    '//div[@id="pager_bottom"]//a[contains(@class, "current")]/following-sibling::a[1]/@href'
).get()
if next_page:
    yield response.follow(next_page, callback=self.parse)
```

**改造后：使用 CrawlSpider（两阶段提取）**

列表页已有 description 和 cover_image，正确做法是：**列表页提取部分字段 → 通过 meta 传递 → 详情页补全剩余字段**。

CrawlSpider 的 Rule 如果直接匹配详情页链接，callback 拿到的是详情页 response，列表页数据就丢了。正确架构是：**Rule 只负责分页跟踪，`parse_start_url` 负责列表页解析和详情页请求的发起**。

```python
# spiders/article_crawl.py
import scrapy
from scrapy.spiders import CrawlSpider, Rule
from scrapy.linkextractors import LinkExtractor
from cnblog.loaders import ArticleLoader
from cnblog.items import ArticleItem
from cnblog.core.cookie_manager import CookieManager
from cnblog.utils.common import article_author, article_info, get_submain, str_datetime


class ArticleCrawlSpider(CrawlSpider):
    name = 'article_crawl'
    allowed_domains = ['www.cnblogs.com', 'news.cnblogs.com']
    start_urls = ['https://www.cnblogs.com/', 'https://news.cnblogs.com/']

    cookieManager = CookieManager()
    cookie = cookieManager.get_random_cookie(name)

    # Rule 只负责分页跟踪，不匹配详情页链接
    rules = (
        # 博客分页：跟踪 pager 中的翻页链接
        Rule(
            LinkExtractor(restrict_css='div#pager_bottom a'),
            follow=True,
        ),
        # 新闻分页：跟踪 pager 中的翻页链接
        Rule(
            LinkExtractor(restrict_css='div.pager a'),
            follow=True,
        ),
    )

    async def parse_start_url(self, response):
        """
        CrawlSpider 对 start_urls 和 follow=True 到达的页面都会调用此方法。
        在这里解析列表页，提取基础字段，手动 yield Request 到详情页。
        """
        parse_url = await get_submain(response.url)
        submain = parse_url["subdomain"]

        if submain == "www":
            for art in response.xpath('//article[@class="post-item"]'):
                loader = ArticleLoader(item=ArticleItem(), selector=art)
                loader.add_xpath('title', './/a[@class="post-item-title"]/text()')
                loader.add_xpath('url', './/a[@class="post-item-title"]/@href')
                loader.add_xpath('description', './/p[@class="post-item-summary"]/text()[last()]')
                loader.add_xpath('cover_image', './/p[@class="post-item-summary"]/a/img/@src')
                loader.add_xpath('created_at', './/span[@class="post-meta-item"]/span/text()')
                item = loader.load_item()

                detail_url = art.xpath('.//a[@class="post-item-title"]/@href').get()
                if detail_url and '/p/' in detail_url:
                    yield response.follow(
                        detail_url,
                        callback=self.parse_detail,
                        meta={'item': item},
                    )

        elif submain == "news":
            for div in response.xpath('//div[@class="news_block"]'):
                loader = ArticleLoader(item=ArticleItem(), selector=div)
                loader.add_xpath('title', './/h2[@class="news_entry"]/a/text()')
                loader.add_xpath('url', './/h2[@class="news_entry"]/a/@href')
                loader.add_xpath('description', './/div[@class="entry_summary"]/text()[last()]')
                loader.add_xpath('cover_image', './/div[@class="entry_summary"]/a/img/@src')
                loader.add_xpath('created_at', './/div[@class="entry_footer"]/span[@class="gray"]/text()')
                item = loader.load_item()

                detail_url = div.xpath('.//h2[@class="news_entry"]/a/@href').get()
                if detail_url and '/n/' in detail_url:
                    yield response.follow(
                        detail_url,
                        callback=self.parse_detail,
                        meta={'item': item},
                        cookies=self.cookie,
                    )

    async def parse_detail(self, response):
        """阶段二：从详情页补全 content, nickname, views"""
        item = response.meta['item']
        parse_url = await get_submain(response.url)
        submain = parse_url["subdomain"]

        if submain == "www":
            content = response.xpath('//div[@class="postBody"]').get()
            if content is None:
                content = response.xpath('//div[@id="cnblogs_post_body"]').get()
            item['content'] = content

            author_prefix = parse_url["path"].split("/")[1]
            author_full_url = f"{parse_url['domain']}/{author_prefix}"
            item['nickname'] = await article_author(author_full_url)
            item['views'] = await article_info(response.url)

        elif submain == "news":
            item['content'] = response.xpath('//div[@id="news_content"]').get()
            item['nickname'] = response.xpath('//span[@class="news_poster"]/a/text()').get()
            item['views'] = await article_info(response.url)

        yield item
```

**为什么这样设计：**

CrawlSpider 的内部流程是：`parse()` → 遍历 Rules → 匹配链接 → 调用 callback 或 follow。

```
start_urls / follow 到达的页面
        │
        ▼
  CrawlSpider.parse()
        │
        ├─ Rule 匹配到分页链接？ → follow=True → 请求新列表页 → 再次进入 parse()
        │
        └─ 没有 Rule 匹配详情页链接 → 调用 parse_start_url()
                │
                ├─ 提取 title, url, description, cover_image, created_at
                └─ yield response.follow(detail_url, callback=parse_detail, meta={'item': item})
                        │
                        ▼
                  parse_detail()
                        ├─ 从 response.meta['item'] 取出已有字段
                        ├─ 补全 content, nickname, views
                        └─ yield item → 进入 Pipeline
```

> **关键点**：如果把详情页链接也写成 Rule + callback，callback 拿到的是详情页 response，列表页的 description 和 cover_image 就丢了。用 `parse_start_url` 处理列表页，手动 yield 带 meta 的 Request，才能把列表页数据传递到详情页。

**字段来源对照：**

| 字段 | 提取阶段 | 提取位置 |
|---|---|---|
| `title` | 列表页 | `//a[@class="post-item-title"]/text()` |
| `url` | 列表页 | `//a[@class="post-item-title"]/@href` |
| `description` | 列表页 | `//p[@class="post-item-summary"]/text()` |
| `cover_image` | 列表页 | `//p[@class="post-item-summary"]/a/img/@src` |
| `created_at` | 列表页 | `//span[@class="post-meta-item"]/span/text()` |
| `content` | 详情页 | `//div[@class="postBody"]` 或 `//div[@id="cnblogs_post_body"]` |
| `nickname` | 详情页 | `article_author()` AJAX 请求 |
| `views` | 详情页 | `article_info()` AJAX 请求 |

### CrawlSpider 的优势

1. **声明式规则**：链接提取和分页逻辑一目了然，不需要手写 XPath 翻页
2. **自动去重**：内置 URL 去重，避免重复爬取
3. **规则组合**：多个 Rule 可以共存，分别处理不同类型的链接
4. **减少样板代码**：分页逻辑从 5-10 行缩减为一行 Rule 声明

### CrawlSpider 注意事项

1. **不要重写 `parse()`**：这是 CrawlSpider 的内部引擎入口，自定义逻辑用其他方法名
2. **Rule 的顺序很重要**：Scrapy 按顺序匹配 Rule，先匹配的先执行
3. **`follow` vs `callback`**：`follow=True` 表示继续从该页面提取链接但不解析数据；`callback` 指定解析方法
4. **与 Playwright 兼容**：CrawlSpider 的 Rule 机制与 scrapy-playwright 的 `meta={"playwright": True}` 可以共存
5. **不适合所有场景**：如果爬取逻辑非常特殊（如需要动态生成请求），普通 Spider + 手动控制可能更灵活

## 总结

当前项目在异步架构、Pipeline 链、数据库批量写入、图片下载、Cookie 管理等方面已达到中高级水平。主要提升方向：

1. **数据提取层**：引入 ItemLoader + Processor，将提取逻辑和清洗逻辑解耦，提升可维护性和可测试性
2. **爬取策略层**：引入 CrawlSpider + LinkExtractor，用声明式规则替代手动分页，减少样板代码

补齐这两块后，再加上代理轮换和 Cookie 自动刷新，就构成了一个完整的生产级爬虫系统。
